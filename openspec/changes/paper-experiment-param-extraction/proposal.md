## Why

科研团队每天阅读大量论文，需要从 PDF/DOCX/Markdown 中手动提取实验参数（温度、压力、收率、催化剂类型等）并整理为结构化数据库。当前完全依赖人工，效率低、易出错、不可追溯。开发一个 AI 辅助的实验参数提取平台，将论文解析、参数提取、人工审核、版本管理、参数库查询全流程线上化，显著提升科研数据整理效率。

## What Changes

- 新增论文上传模块，支持 PDF/DOCX/Markdown 格式，自动解析论文元数据（作者、DOI 等）
- 新增参数模板管理，用户可自定义实验参数模板（如催化剂实验、电池实验模板）
- 新增 AI 参数提取工作流，基于选定模板从论文中自动提取结构化参数（含 JSON 校验与自动重试）
- 新增人工审核与校验流程，支持原文对照、修改/删除/新增参数
- 新增多版本记录，每次修改保存版本并支持差异对比
- 新增参数库，支持搜索、标签、导出 Excel/CSV
- 新增 AI 自然语言问答，基于参数库智能查询
- 新增 Dashboard 统计面板（论文数、模板数、参数数、成功率、趋势图表）
- 新增通知中心（解析完成、提取完成、审核通过/拒绝，未读计数）
- 新增审计日志（谁、什么时间、修改了什么）
- 新增 RBAC 权限系统（普通用户 / 审核员 / 管理员）
- 新增边界处理：模板引用保护、审核后参数归档而非删除、同名文件自动重命名、并发编辑冲突提示

## Capabilities

### New Capabilities
- `paper-upload`: 论文上传与自动解析（PDF/DOCX/Markdown），状态流转（待解析→解析中→解析成功/失败），元数据提取
- `paper-management`: 论文列表管理，支持搜索、筛选、排序，含文件名、上传时间、作者、DOI、状态、提取参数数
- `template-management`: 参数模板的创建、编辑、删除（引用保护）、克隆，YAML 格式定义
- `ai-parameter-extraction`: AI 驱动的参数提取工作流，模板匹配→调用 AI→JSON 校验→非法 JSON 自动重试（最多3次）
- `parameter-review`: 人工审核校验页面，原文与提取结果左右对照，支持修改/删除/新增
- `version-history`: 多版本记录管理，每次修改保存版本，支持差异对比（类 Git Diff）
- `parameter-library`: 最终确认的参数库，支持搜索、标签、导出 Excel/CSV
- `ai-qa`: 自然语言查询参数库，AI 自动转换查询条件并返回结果
- `dashboard`: 统计面板，论文总数/模板总数/参数总数/成功率 + 上传趋势/提取成功率/模板使用排名图表
- `notification-center`: 通知中心，事件驱动通知（解析完成、提取完成、审核通过/拒绝），未读计数
- `audit-log`: 审计日志，记录操作者、时间、变更内容
- `rbac`: 角色权限系统（普通用户可上传/查看/提交审核；审核员可审核/修改结果；管理员拥有全部权限）

## Impact

- 全新全栈 Web 应用，涉及前端、后端、数据库、AI 集成
- 数据库：需设计论文表、模板表、参数表、版本表、用户表、角色权限表、通知表、审计日志表
- AI 集成：需对接 LLM API 进行参数提取和自然语言查询
- 文件存储：需处理 PDF/DOCX/Markdown 文件上传与文本解析
- 前端：Dashboard、论文管理、模板管理、审核页面、参数库、问答、通知中心等页面
- 权限：需实现 RBAC 中间件，角色区分普通用户/审核员/管理员

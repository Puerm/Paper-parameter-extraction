# 参数模板页改版设计稿

## 目标

将“参数模板”页面从当前偏原型感的卡片列表，改造成一个同时兼顾**快速浏览**与**深度查看/编辑**的管理页。改版需要解决以下问题：

- 页面右侧留白过多，信息密度偏低
- 卡片内部信息层级不清，预览区和操作区存在感接近
- 高优先级动作不够突出，编辑/复制/删除的视觉权重接近
- 模板内容只能在卡片里局部查看，不利于深度确认
- 页面缺少“浏览 → 查看详情 → 操作”的清晰交互链路

## 范围

本次改版只覆盖 `frontend/src/pages/Templates.tsx` 所对应的模板页交互与视觉结构，以及为该页服务的详情抽屉行为。

不包含：

- 后端模板接口改造
- 数据模型改造
- 新增搜索、筛选、分页等数据能力
- 模板权限模型改动

## 设计原则

1. **浏览与操作分离**：列表负责扫一眼，抽屉负责看清楚和处理。
2. **主动作突出**：`去提取` 应始终是最明确的主操作。
3. **深度信息不堆在卡片里**：卡片只展示摘要，完整 YAML 放入详情抽屉。
4. **低打断交互**：高频查看不应依赖跳页。
5. **管理员/普通用户分层**：非管理员只看详情和去提取，不暴露编辑类动作。

## 页面结构

### 1. 顶部摘要区

页面顶部保留标题与副标题，并用更明确的视觉层次组织信息：

- 标题：`参数模板`
- 副标题：说明模板用于定义 AI 提取参数字段
- 右侧主按钮：`新建模板`

如果后续需要补充页面统计，可在标题下方追加一行轻量摘要，例如：

- 模板总数
- 参数总数
- 最近更新

本次设计默认保留扩展位，但不强制新增统计卡。

### 2. 工具条

标题区下方增加一条轻量工具条，用于承接未来扩展能力，也用于缓解页面空白感：

- 左侧：搜索入口或筛选入口预留位
- 右侧：轻量辅助信息，如“共 N 个模板”

如果当前没有搜索能力，可先只放统计文本，不强行堆控制项。

### 3. 主内容区

主内容区继续使用卡片，但从“代码块展示卡”调整为“模板摘要卡”：

- 默认两列网格
- 卡片高度更统一
- 标题、元信息、预览、操作区分层清晰
- 点击标题或预览打开详情抽屉

### 4. 详情抽屉

新增右侧抽屉作为深度查看与编辑入口：

- 展示完整模板名称、参数数量、创建/更新时间
- 默认展示完整 YAML 只读预览
- 进入编辑态后，在同一抽屉内切换为可编辑状态
- 底部提供完整操作区：编辑、克隆、删除、去提取

抽屉的定位是“深度查看 + 低打断修改”，不是另一个页面。

## 卡片设计

### 信息层级

每张卡片按以下优先级组织：

1. **标题**
   - 模板名称放在最上方
   - 允许两行展示，避免过早截断
   - 标题可点击，打开详情抽屉

2. **元信息**
   - 参数数量
   - 更新时间
   - 用较弱的视觉样式展示

3. **预览区**
   - 只显示 YAML 的前几行
   - 采用更接近“摘要预览”的样式，而不是大块代码框
   - 预览区同样可点击，打开详情抽屉

4. **操作区**
   - 主操作：`去提取`
   - 次操作：编辑、克隆
   - 危险操作：删除

### 交互规则

- 卡片 hover 仅做轻微抬升或边框强调，不做强装饰动画
- 点击卡片标题/预览进入详情抽屉
- 点击 `去提取` 直接跳转，不进入抽屉
- 编辑、克隆、删除在卡片中保留，但视觉权重要低于主操作
- 删除必须二次确认
- 非管理员不显示编辑、克隆、删除，仅保留详情和去提取入口

## 详情抽屉设计

### 顶部区

- 模板名称
- 参数数量
- 创建时间 / 更新时间
- 操作按钮组：编辑、克隆、删除、去提取

### 主内容区

- 默认显示完整 YAML 预览
- YAML 区域支持滚动
- 在只读和编辑态之间切换时，保持同一内容区域，不跳出抽屉

### 底部提示

- 若模板被引用，展示轻量的保护提示
- 该提示用于降低误删风险

## 空状态

当模板列表为空时，保留一个更完整的空状态卡：

- 图标或插画感轻量表达
- 简短说明：提示用户先创建模板
- 管理员显示 `创建第一个模板`
- 非管理员只显示说明，不显示创建按钮

## 视觉风格

本页应保持现有产品的轻后台风格，但加强层次与一致性：

- 页面整体使用更稳定的内容宽度，避免右侧大片空白
- 卡片边框、阴影、圆角统一
- 主按钮保持高强调度，次级按钮和图标按钮明显降权
- 预览区采用更柔和的背景和边框，和操作区拉开视觉距离

## 交互优先级

从高到低排序如下：

1. `新建模板`
2. `去提取`
3. 打开详情抽屉查看模板
4. 编辑
5. 克隆
6. 删除

这个顺序确保用户先完成最常见任务，再处理管理动作。

## 风险与约束

- 如果抽屉内编辑功能一次性放得太重，页面会接近“编辑页”，失去浏览效率
- 如果卡片预览显示太多，主内容区会重新变得拥挤
- 如果把动作按钮做得太强，视觉焦点会重新散掉

因此本次改版控制原则是：**卡片看摘要，抽屉看详情，主动作最突出**。

## 验收标准

完成后应满足：

- 页面不再出现明显的“空、散、平”感
- 用户可在不跳页的前提下查看完整模板内容
- 主操作 `去提取` 一眼可见
- 管理操作不会和浏览信息抢注意力
- 普通用户不会看到不该有的管理动作

## 建议实现顺序

1. 调整页面主结构与内容宽度
2. 重构模板卡片信息层级
3. 增加详情抽屉
4. 调整按钮优先级与删除确认
5. 补空状态与无权限状态

# PaperParams 前端全面改版设计稿

## 目标

让科研人员在日常使用这个平台时，能感受到愉悦和动力。通过色彩、动效、文案和细节质感，把"又一个科研工具"变成"用这个工具心情会好一点"。

## 设计原则

1. **科研专业感是底线** — 配色大胆但排版干净，不因为活泼而显得业余。
2. **色彩先于装饰** — 用紫罗兰/珊瑚配色系统本身传递情绪，不依赖插画和多余图形。
3. **细节即体验** — 卡片圆角、阴影层次、hover 动效、间距呼吸感，这些"说不清哪里好但就是舒服"的细节。
4. **有梗但克制** — 空状态和 toast 可以有趣，但操作路径本身保持高效。

## 色彩系统

### 主色调
- **主色：** 靛紫 `#6366F1` → 淡紫 `#A78BFA` 渐变
- **强调色：** 珊瑚粉 `#F472B6` → 暖粉 `#FB7185`
- **导航背景：** `linear-gradient(135deg, #6366F1, #8B5CF6)`

### 语义色
- **成功/完成：** 薄荷绿 `#10B981` / `#34D399`
- **警告/待处理：** 琥珀橙 `#F59E0B` / `#FBBF24`
- **错误/拒绝：** 玫瑰红 `#F43F5E` / `#FB7185`
- **信息/进行中：** 天蓝 `#3B82F6` / `#60A5FA`

### 中性色
- 背景：`#F8FAFC`（浅灰蓝）
- 卡片：白色 + `ring-1 ring-slate-200` + `shadow-sm`
- 文本主：`#1E293B` (slate-800)
- 文本次：`#64748B` (slate-500)

## 导航侧栏

- 紫罗兰渐变背景，白色文字
- 当前选中项：珊瑚粉左侧光晕（左边框+微光晕）
- hover：亮度提升 + 轻微右移动画
- 项目管理功能（用户管理、审计日志）用浅色分割线隔开

## 卡片

- 纯白底 + 圆角 `rounded-xl` + 微阴影 `shadow-sm`
- hover 时：`-translate-y-0.5` + `shadow-lg` 抬升效果
- 不同模块卡片通过顶部彩色边框或彩色角标区分（紫罗兰、珊瑚、薄荷、琥珀）

## 按钮系统

| 类型 | 样式 | 交互 |
|------|------|------|
| 主按钮 (primary) | 紫罗兰渐变填充 | hover 轻微放大 + 点击波纹 |
| 次按钮 (secondary) | 珊瑚描边 | hover 填充珊瑚色 |
| 文字按钮 (ghost) | 无背景 | hover 浅色背景 |
| 危险按钮 (destructive) | 玫瑰红描边 | hover 红色背景 |

## 动效

- 页面切换：轻微 `fade-in + translate-y(4px)`，200ms ease-out
- 卡片 hover：150ms `translate-y(-2px)` + shadow 过渡
- 按钮点击：scale(0.97) 回弹 + 波纹效果
- 弹窗/抽屉打开：zoom-in 95→100 + fade，200ms

## 文案风格（有梗部分）

- **加载中：** "正在拼命解析… 咖啡已经准备好了" / "让 AI 再想想… 它也是第一次看这篇论文"
- **加载完成：** "解析完毕 🎉 比你想象中快，对吧？"
- **提取成功：** "参数提取成功 ✨ 比 reviewer #2 快多了" / "搞定 👏  reviewer 要是有这效率就好了"
- **空状态（无模板）：** "还没有模板呢 🧑‍🔬 去建一个吧，比写论文简单多了"
- **空状态（无论文）：** "一篇论文都没有… 是时候治治你的拖延症了 📄"
- **错误：** "翻车了 🫠 要不你再试一次？" / "出错了 🙈 不是你的问题，是代码的问题"
- **删除确认：** "确定删除吗？删了 reviewer 也救不回来 😬"

## 范围

本次改版覆盖：
- `frontend/src/components/Layout.tsx` — 导航渐变背景、交互样式
- `frontend/src/pages/Templates.tsx` — 已经优化了卡片和弹窗，适配新色彩
- `frontend/src/components/ui/` — button.tsx、card.tsx、badge.tsx、dialog.tsx、sheet.tsx 色彩/动效统一
- `frontend/src/pages/*.tsx` — 各页面的空状态和文案
- `frontend/src/index.css` — 全局变量/主题色定义

不包含：
- 后端颜色/样式改造
- 新增页面功能或数据模型
- 暗色模式（本次不做）

## 实现顺序

1. 定义全局 CSS 变量（色彩 token）
2. 改造导航侧栏（渐变背景+交互）
3. 统一按钮/卡片/徽章样式
4. 统一弹窗/抽屉动效
5. 各页面空状态和文案更新
6. 细节收尾：hover 动效、间距、圆角一致性
